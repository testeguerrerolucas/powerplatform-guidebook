Conectores Personalizados

Crie suas próprias integrações

Desenvolva conectores personalizados para APIs específicas do seu negócio.

1

Defina sua OpenAPI

Comece com uma especificação OpenAPI (Swagger) para definir endpoints, parâmetros e respostas.


{
  "swagger": "2.0",
  "info": {
    "title": "API Exemplo",
    "version": "v1"
  },
  "host": "api.exemplo.com",
  "paths": {
    "/products": {
      "get": {
        "responses": {
          "200": {
            "description": "OK"
          }
        }
      }
    }
  }
}
2

Configure a Segurança

Especifique os métodos de autenticação para sua API (OAuth2, Chave API, etc).


{
  "securityDefinitions": {
    "api_key": {
      "type": "apiKey",
      "name": "api_key",
      "in": "header"
    }
  }
}
3

Crie Políticas

Adicione políticas de transformação, validação e manipulação de respostas.



  
    
    
      custom-value
4

Teste e Publique

Teste suas operações e publique o conector para uso em toda a organização.

Exemplos Práticos

Integração na Prática

Veja como usar conectores em cenários reais de negócio.

Integração com CRM

Fluxo de trabalho para sincronização de leads do Salesforce com o Dynamics 365.


// Power Automate Flow - Pseudocódigo
Trigger: Quando um novo lead é criado no Salesforce
Ação 1: Obter detalhes do lead do Salesforce
Ação 2: Verificar se o lead já existe no Dynamics 365
Condição: SE o lead não existe ENTÃO
  Ação 3: Criar novo lead no Dynamics 365
  Ação 4: Adicionar tarefa de acompanhamento
SENÃO
  Ação 5: Atualizar lead existente no Dynamics 365
  Ação 6: Registrar atividade de atualização
FIM Condição
Ação 7: Enviar notificação por email

Integração com ERP

Automatização do processo de aprovação de pedidos de compra com SAP.


// Power Automate Flow - Pseudocódigo
Trigger: Quando um novo pedido de compra excede R$10.000
Ação 1: Consultar detalhes do pedido no SAP
Ação 2: Verificar orçamento disponível
Condição: SE orçamento disponível ENTÃO
  Ação 3: Iniciar fluxo de aprovação
  Ação 4: Esperar aprovação do gerente
  Condição: SE aprovado ENTÃO
    Ação 5: Enviar pedido para processamento no SAP
    Ação 6: Atualizar registro financeiro
  SENÃO
    Ação 7: Notificar solicitante sobre recusa
  FIM Condição
SENÃO
  Ação 8: Rejeitar automaticamente e notificar
FIM Condição

Integração com Mídias Sociais

Monitoramento e resposta automatizada a menções no Twitter.


// Power Automate Flow - Pseudocódigo
Trigger: Quando uma nova menção ao @SuaMarca ocorre no Twitter
Ação 1: Analisar sentimento do tweet usando AI Builder
Condição: SE sentimento é negativo ENTÃO
  Ação 2: Criar caso de suporte no Dynamics 365
  Ação 3: Notificar equipe de atendimento
  Ação 4: Responder com mensagem padrão de suporte
SENÃO SE sentimento é positivo ENTÃO
  Ação 5: Salvar tweet em lista de depoimentos
  Ação 6: Responder agradecendo
  Ação 7: Verificar se é cliente existente
SENÃO
  Ação 8: Categorizar para análise posterior
FIM Condição
Ação 9: Registrar interação no banco de dados

Integração com API Personalizada

Conexão com sistema legado interno usando conector personalizado.


// Definição do Conector Personalizado
{
  "swagger": "2.0",
  "info": {
    "title": "Sistema Legado API",
    "version": "v1"
  },
  "host": "api.interno.empresa.com",
  "basePath": "/v1",
  "schemes": ["https"],
  "paths": {
    "/clientes/{id}": {
      "get": {
        "summary": "Obter Cliente",
        "parameters": [{
          "name": "id",
          "in": "path",
          "required": true,
          "type": "string"
        }],
        "responses": {
          "200": {
            "description": "OK"
          }
        }
      }
    }
  }
}

// Uso no Power Apps (Pseudocódigo)
função ObterDadosCliente(id) {
  resposta = ConectorLegado.ObterCliente(id);
  return resposta;
}
